# 에이전트 구성표

> 이 프로젝트는 5명의 전문 에이전트가 협업하여 개발한다.
> 각 에이전트는 자기 역할 문서를 따르고, 다른 에이전트 파일을 참조하며 일한다.

---

## 에이전트 목록

| # | 파일 | 역할 | 담당 |
|---|------|------|------|
| 1 | `agent_db.md` | DB 에이전트 | 데이터베이스 설계, 모델, 시드 데이터 |
| 2 | `agent_backend.md` | 백엔드 에이전트 | FastAPI 서버, API, 비즈니스 로직 |
| 3 | `agent_frontend.md` | 프론트엔드 에이전트 | Jinja2 템플릿, HTMX, Alpine.js, Tailwind |
| 4 | `agent_qa.md` | 검수 에이전트 | 테스트 작성/실행, 버그 검출, 코드 리뷰 |
| 5 | `agent_security.md` | 보안 에이전트 | 보안 점검, 취약점 탐지, 최종 감사 |

---

## 작업 순서

```
Phase 1: DB 에이전트        → 테이블 설계, 모델 생성, 시드 데이터
Phase 2: 백엔드 에이전트     → API 서버, 라우터, 서비스 로직
Phase 3: 프론트엔드 에이전트  → 템플릿 분리, HTMX 연동, 화면 완성
Phase 4: 검수 에이전트       → 단위 테스트 + E2E 테스트 + 버그 수정
Phase 5: 보안 에이전트       → 보안 감사 + 취약점 리포트 + 최종 확인
```

## 참고 코드베이스

`kevinsaem_v31_dev/` 폴더에 이미 동작하는 참고 코드가 있다. 각 에이전트는 자기 영역의 참고 코드를 반드시 읽고 패턴을 활용한다.

| 참고 파일 | 참고하는 에이전트 |
|-----------|-----------------|
| `kevinsaem_v31_dev/app/seed.py`, `database.py`, `models/` | DB 에이전트 |
| `kevinsaem_v31_dev/main.py`, `routers/`, `services/`, `config.py` | 백엔드 에이전트 |
| `kevinsaem_v31_dev/templates/`, `static/` | 프론트엔드 에이전트 |

> **단, 프론트엔드 디자인의 기본은 `케빈샘_완전동작_프로토타입_v3.html`이다.**
> v31_dev 템플릿은 구조 참고용이지, 디자인 원본이 아니다.

## 소통 규칙

1. **작업 전** — 관련 에이전트 파일을 반드시 읽는다
2. **작업 후** — 자기 파일의 `현재 상태`와 `산출물` 섹션을 업데이트한다
3. **다른 에이전트에 영향** — 해당 에이전트 파일의 `인바운드 알림` 섹션에 메모를 남긴다
4. **일관성 체크** — DB 스키마 ↔ API 타입 ↔ 프론트엔드 기대값이 항상 일치해야 한다
5. **충돌 발생 시** — 사장에게 보고하고 판단을 기다린다
