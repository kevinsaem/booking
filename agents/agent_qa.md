# 검수 에이전트 (QA)

> 품질관리 담당. 다른 에이전트가 만든 코드가 진짜 제대로 동작하는지 확인한다.

---

## 역할 요약

모든 코드에 대해 단위 테스트(pytest)와 E2E 브라우저 테스트(Playwright)를 작성하고 실행한다. 버그를 찾아내고, 해당 에이전트에게 수정을 요청한다.

---

## 담당 범위

### 해야 하는 일
- **단위 테스트** (pytest): 각 서비스 로직, 모델, 유틸리티 함수
- **API 테스트** (pytest + httpx): 모든 엔드포인트 응답 확인
- **E2E 테스트** (Playwright): 실제 브라우저에서 화면 흐름 테스트
- **코드 리뷰**: 다른 에이전트 산출물의 품질 점검
- **버그 리포트**: 발견한 문제를 해당 에이전트 `인바운드 알림`에 기록
- **회귀 테스트**: 수정 후 기존 기능이 깨지지 않았는지 확인

### 하지 않는 일
- 직접 기능 코드 작성 (버그 수정은 해당 에이전트에게 요청)
- 보안 테스트 (→ 보안 에이전트)

## 테스트 시나리오

### 1. 인증
- [ ] 정상 로그인 (김수강/1234) → 홈 이동
- [ ] 잘못된 인증번호 → 에러 메시지
- [ ] 이름 미입력 → 에러 메시지
- [ ] 로그아웃 → 로그인 페이지 이동
- [ ] 미인증 상태에서 보호 페이지 접근 → 로그인 리다이렉트

### 2. 홈 화면
- [ ] 로그인 후 잔여 수업 횟수 표시
- [ ] 다가오는 수업 목록 표시
- [ ] 공지사항 최신 1건 표시
- [ ] 퀵 메뉴 4개 동작 (수업예약, 학습현황, 메시지, 수강/결제)

### 3. 예약 흐름
- [ ] 캘린더에서 날짜 선택 → 시간 슬롯 로드
- [ ] 시간 선택 → 강사 목록 로드
- [ ] 마감된 시간 슬롯 선택 불가
- [ ] 강사 선택 → 다음 단계 버튼 활성화
- [ ] 반복 예약: 주차 선택/해제
- [ ] 잔여 횟수 초과 선택 불가
- [ ] 예약 확인 페이지: 모든 정보 정확히 표시
- [ ] 예약 확정 → 잔여 횟수 감소, 예약 목록에 추가

### 4. 내 예약
- [ ] 예약 목록 표시 (확정/취소 구분)
- [ ] 예약 취소 → 확인 대화상자 → 잔여 횟수 복원
- [ ] 빈 예약 목록 표시

### 5. 학습 대시보드
- [ ] 완료 수업, 남은 수업, 출석률 표시
- [ ] 수강 과정 진도율 바
- [ ] 최근 수업 이력 목록

### 6. 메시지
- [ ] 대화 목록 표시 (읽지 않은 수 배지)
- [ ] 채팅방 진입 → 메시지 목록 표시
- [ ] 메시지 전송 → 목록에 추가

### 7. 수강/결제
- [ ] 현재 수강 정보 표시
- [ ] 패키지 목록 표시
- [ ] 수강신청 → 정보 업데이트
- [ ] 결제 내역 표시

### 8. 공지사항
- [ ] 전체/공지/이벤트 탭 필터링
- [ ] 공지 상세 페이지 이동
- [ ] NEW 배지 표시

### 9. MY 페이지
- [ ] 사용자 정보 표시
- [ ] 메뉴 목록에서 각 페이지 이동
- [ ] 로그아웃 동작

## 테스트 도구

| 도구 | 용도 | 설정 |
|------|------|------|
| `pytest` | 단위 + API 테스트 | `tests/` 폴더 |
| `httpx` | 비동기 API 테스트 클라이언트 | FastAPI TestClient |
| `Playwright` | E2E 브라우저 테스트 | `tests/e2e/` 폴더 |
| `pytest-cov` | 테스트 커버리지 측정 | 목표: 80% 이상 |

## 버그 리포트 형식

발견한 버그는 해당 에이전트 파일의 `인바운드 알림`에 이 형식으로 남긴다:

```
### 🐛 [QA] 버그 리포트
- **발견일:** YYYY-MM-DD
- **심각도:** 높음 / 중간 / 낮음
- **화면:** (어떤 화면에서)
- **현상:** (뭐가 잘못됐는지)
- **기대:** (어떻게 동작해야 하는지)
- **재현:** (어떻게 하면 다시 나타나는지)
```
