## 1. 학습 현황 — 수강권 상세 카드 개선

### 현재 상태
수강권 선택 후 아래에 과정명과 완료된 수업 횟수만 표시됨.

### 변경 요구사항
수강권 선택 시 하단 카드 영역에 다음 3가지 항목을 **분리하여** 표시:

| 항목 | 설명 |
|------|------|
| 총 배정 횟수 | 해당 수강권에 포함된 전체 수업 횟수 |
| 예약 중인 수업 | 현재 예약이 잡혀 있는 수업 횟수 |
| 완료된 수업 | 이미 수강 완료된 수업 횟수 |

- **잔여 횟수**도 함께 계산하여 표시: `잔여 = 총 배정 - 예약 중 - 완료`
- 카드 UI에서 한눈에 진행 상황을 파악할 수 있도록 구성

### 수업 기록 섹션
- 현재: 슬라이딩(캐러셀) 형식
- 유지 가능: 각 완료/예약 건별로 카드 형태로 표시하되, 추후 **교사 리포트** 및 **수강생 리포트**가 연결될 예정이므로 개별 카드 구조가 적합

## 2. 수업 예약 시 과정 유형별 분리 (일반 과정 vs 성인 1:1 과정)

### 배경
- ~~현재 수강생 타입(MBR type)으로 성인/일반을 구분하고 있었으나~~ **개념 변경됨**
- **수강권(과정) 기준**으로 분리하며, 수강생의 `mem_mbrtype`은 성인/학생 구분 없이 **일괄 `2`로 고정**
- 한 수강생이 일반 과정과 성인 1:1 과정을 **동시에** 보유할 수 있음

### DB 스키마 기준 정보

#### 과목 정보
- **`ek_wb` 테이블**: `wb_idx = 256` → 성인 1:1 과정 과목 정보

#### 성인 1:1 상품 (총 6종) 조회
```sql
SELECT * FROM ek_package WHERE state = 1 AND class_cnt = 1;
```
- `state = 1`: 활성 상품
- `class_cnt = 1`: 1:1 수업 상품 (주 1회 단위)
- 이 쿼리 결과가 성인 1:1 과정 6종에 해당

#### 회원 타입 (`ek_member.mem_mbrtype`) — ⚠️ 중요 변경

| mem_mbrtype | 역할 | 비고 |
|-------------|------|------|
| `2` | **수강생 (통합)** | 성인/학생 구분 없이 모든 수강생은 2로 고정 |
| `4` | **1:1 수업 강사** | ⚠️ 기존: 성인 수강생 → **변경: 1:1 강사로 재정의** |
| `5` | **일반 강사** | 기존 유지 |

### 변경 요구사항

#### 2-1. 회원가입 코드 수정 (⚠️ 필수)
- `ek_member` INSERT 시 **`mem_mbrtype = 2`로 고정** 입력
- 기존 코드에서 `mem_mbrtype = 4`로 입력하던 부분을 **모두 `2`로 변경**
- 성인 수강생도 일반 수강생과 동일하게 `mem_mbrtype = 2`

#### 2-2. 강사 구분 — 기존 `mem_mbrtype` 활용 (별도 컬럼 불필요)
- `mem_mbrtype = 4` → **1:1 수업 강사**
- `mem_mbrtype = 5` → **일반 강사**
- 강사 목록 조회 시 이 값으로 필터링

#### 과정 구분 — 테이블 연결 경로

```
ek_settlement.package_code  →  ek_package (JOIN)  →  ek_package.class_cnt 로 과정 유형 판별
```

- `ek_settlement` 테이블의 `package_code` 컬럼으로 `ek_package` 테이블을 참조
- `ek_package.class_cnt = 1` → 1:1 과정 상품
- `ek_package.class_cnt > 1` (또는 1이 아닌 값) → 일반 과정 상품

#### 2-4. 수업 예약 플로우 변경
1. 수강생이 **예약하기** 진입
2. 현재 보유한 수강권(잔여 횟수가 있는 것) 목록 표시
   - `ek_settlement` → `ek_package` JOIN하여 각 수강권의 `class_cnt` 확인
3. 선택한 수강권의 **`ek_package.class_cnt`** 값에 따라:
   - `class_cnt = 1` (1:1 과정) → `mem_mbrtype = 4`인 **1:1 강사**의 빈 수업방만 노출
   - `class_cnt ≠ 1` (일반 과정) → `mem_mbrtype = 5`인 **일반 강사**의 빈 수업방만 노출
4. 여러 과정의 수강권을 동시 보유 시, 각각 구분하여 예약 가능하도록 UI 구성

#### 2-5. 수업방 정원 로직 (⚠️ 중요)

| 구분 | 강사 타입 | 정원 | 예약 규칙 |
|------|-----------|------|-----------|
| 1:1 수업방 | `mem_mbrtype = 4` | **1명** | 수강생 1명이 예약하면 해당 수업방은 **즉시 마감** (다른 수강생 예약 불가) |
| 일반 수업방 | `mem_mbrtype = 5` | **제한 없음** | 정원 제한 없이 여러 수강생이 자유롭게 예약 가능 |

**구현 시 주의사항:**
- 1:1 수업방: 예약 시 해당 시간 슬롯의 현재 예약 수를 체크하여, **1명 이상이면 예약 불가** 처리
- 일반 수업방: **정원 제한 없음** — 별도 인원 체크 불필요, 빈 수업방이면 누구나 예약 가능
- 동시 예약 방지를 위한 **동시성 제어**(트랜잭션 또는 락) 고려 필요 — 특히 1:1 수업방에서 중복 예약 방지 필수

## 5. 메시지 기능 — 수강생↔멘토 1:1 메시지 시스템

### 전체 플로우

```
[수강생] 멘토에게 1:1 메시지(질문) 작성/전송
        ↓
[시스템] 담당 멘토의 이메일로 알림 발송 (발신: kevin@kevinsaem.com)
        - 수강생 이름, 메시지 내용(질문) 요약 포함
        - 답변 페이지 직접 링크 포함 (토큰 내장 URL)
        ↓
[멘토] 이메일 수신 → 링크 클릭 → 답변 페이지 바로 노출 (로그인 불필요)
        - 수강생 질문 확인
        - 이전 메시지 히스토리 확인
        - 수강생 학습 이력 조회
        - 텍스트 답변 작성 + 이미지/파일 첨부
        - 답변 등록
        ↓
[시스템] 답변을 수강생의 메시지함에 표시
        + 수강생에게 이메일 알림 발송 ("멘토가 답변을 등록했습니다")
        ↓
[수강생] 메시지 화면에서 멘토의 답변 확인
```

### 5-1. 멘토 → 수강생 알림 이메일 (수강생이 질문 보낼 때)

| 항목 | 내용 |
|------|------|
| 발신 주소 | `kevin@kevinsaem.com` |
| 수신 주소 | 해당 멘토의 `ek_member` 등록 이메일 |
| 메일 제목 | 예) `[케빈샘AI코딩학원] {수강생이름}님이 질문을 보냈습니다` |
| 메일 본문 | 수강생 이름, 질문 내용 요약, **답변하기 링크** (토큰 포함 URL) |

### 5-2. 토큰 기반 답변 페이지 접근

- 이메일 내 링크 형식: `https://{도메인}/message/reply?token={UUID}`
- 토큰 클릭 시 **로그인 없이** 바로 답변 폼 페이지로 이동
- 토큰 유효 기간: 48시간 권장 (만료 시 "링크가 만료되었습니다. 로그인하여 답변해주세요." 안내)
- 토큰은 **1회 답변 완료 후에도 유효 기간 내 재접근 가능** (수정/추가 답변 가능)

### 5-3. 멘토 답변 페이지 — 화면 구성 (⚠️ 핵심)

멘토가 토큰 링크를 클릭하면 아래 요소가 **한 페이지에** 표시되어야 함:

#### A. 상단 — 수강생 정보 영역
- 수강생 이름
- 현재 수강 중인 과정명

#### B. 메시지 히스토리 영역
- 해당 수강생과 이 멘토 간의 **이전 메시지 대화 내역** 전체 표시 (시간순)
- 각 메시지에 발신자(수강생/멘토), 발송 시간 표시
- 현재 질문은 **하이라이트** 처리하여 어떤 질문에 대한 답변인지 명확히 구분

#### C. 수강생 학습 이력 조회 영역
- 접기/펼치기(아코디언) 또는 탭 형태로 제공
- 해당 수강생의 학습 현황 요약: 수강권별 총 배정/완료/잔여 횟수
- 최근 수업 기록 목록 (날짜, 과정명, 담당 강사)

#### D. 답변 작성 영역
- **텍스트 입력**: 여러 줄 입력 가능한 textarea
- **파일/이미지 첨부**: 파일 업로드 버튼 (이미지, PDF 등)
  - 첨부 파일 미리보기 지원
  - 파일 크기 제한 설정 필요 (예: 10MB)
- **답변 등록 버튼**: 클릭 시 답변 저장 + 수강생에게 알림 발송
- **등록 완료 시**: "답변이 등록되었습니다" 확인 메시지 표시

#### E. 반응형 대응
- 모바일/PC 모두 지원 (반응형 레이아웃 필수)
- 모바일에서는 학습 이력 영역을 기본 접힘 상태로 처리

### 5-4. 수강생 → 멘토 답변 완료 알림 이메일

| 항목 | 내용 |
|------|------|
| 발신 주소 | `kevin@kevinsaem.com` |
| 수신 주소 | 해당 수강생의 `ek_member` 등록 이메일 |
| 메일 제목 | 예) `[케빈샘AI코딩학원] {멘토이름} 멘토가 답변을 등록했습니다` |
| 메일 본문 | 멘토 이름, 답변 내용 요약, **메시지 확인하기 링크** (수강생 메시지 화면으로 이동) |

### 5-5. DB 설계 참고

```sql
-- 메시지 테이블
CREATE TABLE ek_message (
    msg_idx         INT AUTO_INCREMENT PRIMARY KEY,
    sender_idx      INT NOT NULL,          -- ek_member.mem_idx (발신자)
    receiver_idx    INT NOT NULL,          -- ek_member.mem_idx (수신자)
    content         TEXT NOT NULL,          -- 메시지 본문
    parent_msg_idx  INT DEFAULT NULL,       -- 스레드: 원본 메시지 ID (NULL이면 최초 질문)
    is_read         TINYINT DEFAULT 0,      -- 0: 미읽음, 1: 읽음
    created_at      DATETIME DEFAULT NOW(),
    updated_at      DATETIME DEFAULT NOW()
);

-- 메시지 첨부파일 테이블
CREATE TABLE ek_message_file (
    file_idx        INT AUTO_INCREMENT PRIMARY KEY,
    msg_idx         INT NOT NULL,           -- ek_message.msg_idx
    file_name       VARCHAR(255) NOT NULL,  -- 원본 파일명
    file_path       VARCHAR(500) NOT NULL,  -- 서버 저장 경로
    file_size       INT DEFAULT 0,          -- 파일 크기 (bytes)
    file_type       VARCHAR(50),            -- MIME type
    created_at      DATETIME DEFAULT NOW()
);

-- 답변 토큰 테이블
CREATE TABLE ek_message_token (
    token_idx       INT AUTO_INCREMENT PRIMARY KEY,
    token           VARCHAR(128) NOT NULL UNIQUE,  -- UUID 토큰
    msg_idx         INT NOT NULL,           -- 대상 메시지
    mentor_idx      INT NOT NULL,           -- 멘토 mem_idx
    expires_at      DATETIME NOT NULL,      -- 만료 시간
    is_used         TINYINT DEFAULT 0,      -- 사용 여부 (0: 미사용)
    created_at      DATETIME DEFAULT NOW()
);
```
