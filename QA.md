# QA 체크리스트 - 케빈샘랩 예약 시스템

## 테스트 대상: https://booking.kevinsaem.com

---

## 1. 랜딩페이지 (/)
- [ ] 페이지 정상 로드
- [ ] 로그인 버튼 → /booking/ 이동
- [ ] 상담신청 플로팅 버튼 → /survey 새창 열기
- [ ] 카카오맵 iframe 정상 표시 (모바일에서 카카오맵 앱으로 이동 안 함)
- [ ] 모바일 반응형 레이아웃

## 2. 설문 페이지 (/survey)
- [ ] 인트로 화면 배경 전체 채움
- [ ] 13개 질문 슬라이드 정상 동작
- [ ] 이름/연락처 입력 필드 동작
- [ ] 요일/시간대 별도 화면
- [ ] 제출 → 구글시트 저장
- [ ] 제출 → 이메일 알림

## 3. 로그인 (/booking/)
- [ ] 로그인 폼 표시 (카카오 버튼 숨김)
- [ ] 이름 + 비밀번호(인증코드) 로그인
- [ ] 비밀번호 MD5 해시로 비교
- [ ] 로그인 실패 시 에러 메시지
- [ ] brute-force 방지 (5회 초과 시 15분 차단)
- [ ] 회원가입 버튼 → 캠퍼스 선택 표시

## 4. 회원가입 (/booking/signup)
- [ ] 캠퍼스 선택 (선부/초지) → edc_idx 전달
- [ ] 정보 입력 (이름, 이메일, 닉네임, 연락처)
- [ ] 아이디 중복 확인
- [ ] 인증코드 발송 (슈어엠 알림톡)
- [ ] 인증코드 4자리 입력 시 자동 검증
- [ ] mem_pwd에 MD5 해시로 저장
- [ ] corp_idx=4, mem_MbrType=4(수강생) 저장
- [ ] edc_idx 정상 저장
- [ ] 가입 완료 후 자동 로그인 → 홈 이동

## 5. 마이페이지 (/booking/mypage)
- [ ] 프로필 카드 표시
- [ ] 개인정보 (이름, 아이디, 닉네임, 연락처, 가입일)
- [ ] 메뉴 링크 동작
- [ ] 로그아웃

## 6. DB 연결
- [ ] DB_MODE = production (Render 환경)
- [ ] MS-SQL 연결 정상
- [ ] SQLite datetime → MS-SQL GETDATE() 변환
- [ ] LIMIT → TOP 변환
- [ ] dev_ 테이블 쿼리 실패 시 빈 값 반환 (사이트 안 죽음)

## 7. 보안
- [ ] .env 파일 git에 미포함
- [ ] JWT 토큰 HttpOnly 쿠키
- [ ] production 모드에서 secure=True
- [ ] 비밀번호 MD5 해시 저장
- [ ] SQL Injection 방지 (파라미터 바인딩)
