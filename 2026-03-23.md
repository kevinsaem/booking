3. **ek_Sch_Detail** — 시간 슬롯 (5분 단위, sch_detail_code, sch_detail_Stime, sch_detail_Etime, sch_room_idx)
4. **ek_Sch_Detail_Room** — 수업방 (연속 슬롯 묶음, sch_room_idx, sch_teach_id, sch_detail_Stime/Etime, mem_grade)
5. **ek_Sch_Detail_Room_mem** — 학생 배정/예약 (idx, sch_room_idx, mem_mbrid, settle_code, l_s_date, l_f_date, status, absent, makeupidx, pm_status, wmbrid)

### 핵심 비즈니스 로직:

**수업방 생성 (gubun=schjoin):**
- 체크박스로 선택한 연속 시간 슬롯들을 하나의 수업방으로 묶음
- ek_Sch_Detail_Room에 INSERT (max(sch_room_idx)+1)
- ek_Sch_Detail에 sch_room_idx UPDATE (선택한 슬롯들 + 같은 요일/시간의 미래 슬롯들 자동 배정)

**수업방 삭제 (gubun=del):**
- ek_Sch_Detail_Room: sch_room_status=2로 UPDATE (soft delete)
- ek_Sch_Detail_Room_mem: status=2로 UPDATE
- ek_Sch_Detail: sch_room_idx=null로 UPDATE (슬롯 해제)

**학생 배정 (ind_sch_2.asp, gubun=in1sert):**
- ek_Sch_Detail_Room_mem에 INSERT (sch_room_idx, mem_mbrid, settle_code, l_s_date, l_f_date, status=1)
- 최대 12주 추가 날짜 체크박스로 일괄 등록 가능
- 보강(makeupidx) 연결 가능 → pm_status=1로 보강 완료 처리

**출결 관리 (schedule.asp):**
- absent 값 순환: 0(사전결석) → 1(출석) → 2(결석) → 0(사전결석)
- 전체 사전결석 일괄 처리 기능 (gubun=allpreabsent)

**결제/수강권 (settlement.asp):**
- ek_Settlement + ek_CashCode + ek_Package JOIN
- settle_state: 0=대기, 1=완료, 2=오류
- settle_gubun: 0=무통장, 1=카드현장, 2=카드전화
- 배정된 스케줄 있으면 삭제 불가 (비즈니스 규칙)

**시간대 처리:**
- 모든 날짜/시간에 GMT+9 변환: dateadd(hour,9,datetime)
- ek_Timezone 테이블로 사용자별 시간대 관리

**좌석 점유율 (dschedule.asp):**
- 20분 단위(00분, 20분, 40분)로 좌석 사용량 계산
- ek_EduCenter.seatnum과 비교하여 초과 시 빨간색 표시
- etc3=0(PC 사용), etc3=1(노PC) 구분

### Python 시스템과의 주요 차이점:
1. **스케줄 계층 누락:** ASP는 5단계(Sch→Day→Detail→Room→Room_mem), Python은 2단계(Room→Room_mem)만 있음
2. **absent/출결:** Python에 absent, makeupidx, pm_status 컬럼 없음
3. **mem_grade:** Python에 mem_grade(학년/대상연령) 없음
4. **ek_CashCode:** Python에 없음 (settle→package 직접 연결)
5. **시간대 변환:** ASP는 UTC 저장 + 조회 시 +9시간, Python은 로컬 시간 직접 저장
6. **sp_gubun:** 특강 구분 필드 없음
7. **demo_gubun:** 체험 수업 구분 없음
8. **settle_code 기반 수업 관리:** ASP는 settle_code로 수강권별 수업을 추적하는 것이 핵심
