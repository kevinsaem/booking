# 작업지시서: 랜딩페이지 접속통계 관리자 메뉴

> 작성: 랜딩페이지 프로젝트(kevinsaem-booking) 에이전트, 2026-07-21
> 대상: 관리자 시스템 프로젝트 에이전트
> 목표: 관리자 사이트에 "접속 통계" 메뉴를 추가하여, 랜딩페이지별 방문 통계를 볼 수 있게 한다.

---

## 1. 배경 — 데이터는 이미 쌓이고 있다 (수집은 이 문서 범위 아님)

랜딩페이지 프로젝트(booking.kevinsaem.com 서버)가 방문자를 자동으로 기록한다.

- 방문자가 랜딩페이지를 열 때마다 **`ek_PageView` 테이블에 1건씩 저장** (MS-SQL, 기존 `ek_Member` 등과 같은 DB)
- **봇/크롤러는 수집 단계에서 이미 걸러진다** (Googlebot, 네이버 Yeti, curl 등 User-Agent 기반). 조회할 때 봇 필터링을 따로 할 필요 없음.
- 성공 응답(200)의 GET 요청만 기록. 시각은 서버 로컬시간(KST).
- 테이블은 랜딩 서버가 시작될 때 자동 생성된다. 관리자 프로젝트에서는 **조회(SELECT)만** 하면 된다. INSERT/DDL 금지.
- **DB 접근**: `ek_PageView`는 랜딩(booking) 프로젝트가 쓰는 MS-SQL DB에 있다. 관리자 시스템이 이미 같은 DB를 쓰고 있다면 추가 설정 없이 바로 조회 가능. 다른 DB를 쓰고 있다면 사장님께 booking DB 접속정보를 요청해 읽기 전용으로 연결할 것 (접속정보는 `.env`에만 저장, 코드/문서에 넣지 말 것).
- 데이터는 랜딩 프로젝트가 실서버에 배포된 시점부터 쌓인다 (과거 소급 없음).

## 2. 테이블 스키마 (MS-SQL)

```sql
CREATE TABLE ek_PageView (
    pv_id INT IDENTITY(1,1) PRIMARY KEY,
    pv_path VARCHAR(100) NOT NULL,          -- 방문 경로 (예: /youth)
    pv_referrer NVARCHAR(500) DEFAULT '',   -- 유입 출처 (Referer 헤더, 직접 방문이면 '')
    pv_user_agent NVARCHAR(500) DEFAULT '', -- 브라우저 정보
    pv_ip VARCHAR(45) DEFAULT '',           -- 방문자 IP
    created_at DATETIME2 DEFAULT GETDATE()  -- 방문 시각 (KST)
);
-- 인덱스: IX_PageView_path_date ON (pv_path, created_at)
```

## 3. 경로 → 표시 이름 매핑

| pv_path | 화면 표시 이름 |
|---|---|
| `/` | 홈 (메인) |
| `/youth` | 유소년 코딩 |
| `/adult` | 성인 코딩 |
| `/autobiography` | 자서전 쓰기 |
| `/corporate` | 기업 교육 |
| `/corporate-survey` | 기업 설문 |
| `/campus` | 캠퍼스 안내 |
| `/tuition/youth` | 유소년 수강료 |
| `/tuition/corporate` | 기업 수강료 |
| `/privacy` | 개인정보처리방침 |
| `/refund` | 환불 규정 |

※ 매핑에 없는 새 경로가 나타나면 경로 그대로 표시하면 된다 (랜딩 프로젝트에서 페이지가 추가될 수 있음).

## 4. 만들 화면 (요구사항)

관리자 사이드바에 **"접속 통계"** 메뉴 추가 (관리자 권한 필수 — 기존 관리자 메뉴와 동일한 인증 적용). 화면 구성 4개 섹션:

1. **요약 카드 4개**: 오늘 / 최근 7일 / 최근 30일 / 전체 누적 방문 수
2. **최근 14일 일별 방문 막대그래프** (방문 없는 날은 0으로 표시)
3. **페이지별 방문 표**: 페이지명, 경로, 오늘/7일/30일/전체 방문 수 (30일 기준 내림차순)
4. **외부 유입 경로 TOP 10** (최근 30일, 자체 사이트 내 이동 제외)

데이터가 하나도 없을 때는 "아직 기록된 방문이 없습니다" 안내 문구를 보여준다.

## 5. 참고 쿼리 (MS-SQL, 검증 완료)

날짜 파라미터는 `'YYYY-MM-DD'` 문자열로 넘기면 된다 (오늘, 6일 전, 29일 전, 13일 전).

```sql
-- ① 페이지별 방문 수 (오늘/7일/30일/전체)
SELECT pv_path,
       SUM(CASE WHEN created_at >= @today THEN 1 ELSE 0 END) as cnt_today,
       SUM(CASE WHEN created_at >= @d7    THEN 1 ELSE 0 END) as cnt_7d,
       SUM(CASE WHEN created_at >= @d30   THEN 1 ELSE 0 END) as cnt_30d,
       COUNT(*) as cnt_total
FROM ek_PageView
GROUP BY pv_path
ORDER BY cnt_30d DESC;
-- 요약 카드는 이 결과를 합산하면 된다.

-- ② 최근 14일 일별 방문 수
SELECT CONVERT(varchar(10), created_at, 23) as day, COUNT(*) as cnt
FROM ek_PageView
WHERE created_at >= @d14
GROUP BY CONVERT(varchar(10), created_at, 23)
ORDER BY day ASC;
-- 결과에 없는 날짜는 0으로 채워서 14일 전부를 그린다.

-- ③ 외부 유입 경로 TOP 10 (최근 30일)
SELECT TOP 10 pv_referrer, COUNT(*) as cnt
FROM ek_PageView
WHERE created_at >= @d30
  AND pv_referrer <> ''
  AND pv_referrer NOT LIKE '%kevinsaem%'   -- 자체 사이트 내 이동 제외
GROUP BY pv_referrer
ORDER BY cnt DESC;
```

## 6. 주의사항

- `pv_ip`, `pv_user_agent`는 개인정보 성격이 있으므로 화면에 노출하지 말 것 (집계 수치만 표시).
- `ek_PageView`에는 **읽기만** 한다. 쓰기/스키마 변경은 랜딩 프로젝트 담당.
- 완성 검증 기준: 랜딩페이지(booking.kevinsaem.com)를 브라우저로 몇 번 방문한 뒤, 관리자 화면 숫자가 그만큼 늘어나는지 확인.
