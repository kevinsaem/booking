# 작업지시서: 랜딩페이지 접속통계 관리자 메뉴 (v2)

> 작성: 랜딩페이지 프로젝트(kevinsaem-booking) 에이전트, 2026-07-21
> 대상: 관리자 시스템 프로젝트 에이전트
> 목표: 관리자 사이트에 "접속 통계" 메뉴를 만들어, 랜딩페이지별 방문 수와 **페이지별 외부 유입경로**를 보여준다.
> 용도: 사장님이 **마케팅(광고/홍보)을 집행한 뒤 그 반응을 확인**하는 화면이다. 이 관점으로 설계할 것.

---

## 1. 배경 — 데이터는 랜딩 서버가 자동 수집한다

- 랜딩페이지 프로젝트(booking.kevinsaem.com)가 방문자를 **`ek_PageView` 테이블에 1건씩 기록**한다.
- **DB는 관리자 시스템과 동일한 MS-SQL DB** (사장님 확인 완료). 별도 접속 설정 없이 바로 조회 가능.
- **봇/크롤러는 수집 단계에서 이미 걸러진다** (Googlebot, 네이버 Yeti, curl 등). 조회 시 봇 필터링 불필요.
- 성공 응답의 GET 요청만 기록. 시각은 서버 로컬시간(KST).
- 테이블·인덱스는 랜딩 서버가 시작될 때 자동 생성된다. 관리자 프로젝트는 **조회(SELECT)만** 한다. INSERT/DDL 금지.
- 데이터는 랜딩 프로젝트가 실서버에 배포된 시점부터 쌓인다 (과거 소급 없음).

## 2. 수집 항목 (ek_PageView 테이블, 방문 1건 = 1행)

| 컬럼 | 타입 | 내용 | 활용 |
|---|---|---|---|
| `pv_id` | INT (PK) | 자동 증가 번호 | |
| `pv_path` | VARCHAR(100) | 방문 경로 (예: `/youth`) | 페이지별 집계 |
| `pv_query` | NVARCHAR(500) | 쿼리스트링 원본 (utm 포함 전체) | 상세 분석 예비용 |
| `pv_referrer` | NVARCHAR(500) | 유입 출처 URL 원본 (Referer) | 상세 확인용 |
| `pv_ref_host` | VARCHAR(100) | **유입 출처 도메인** (www. 제거, 예: `google.com`, `instagram.com`). 직접 방문이면 `''` | 유입경로 집계 (이 컬럼으로 GROUP BY) |
| `pv_utm_source` | NVARCHAR(100) | 마케팅 링크의 `utm_source` (예: `instagram`) | **캠페인 성과 집계** |
| `pv_utm_medium` | NVARCHAR(100) | `utm_medium` (예: `ad`, `post`) | |
| `pv_utm_campaign` | NVARCHAR(200) | `utm_campaign` (예: `2026_summer`) | |
| `pv_device` | VARCHAR(10) | `mobile` / `tablet` / `pc` | 기기별 집계 |
| `pv_user_agent` | NVARCHAR(500) | 브라우저 정보 원본 | 화면 노출 금지 |
| `pv_ip` | VARCHAR(45) | 방문자 IP | 순방문자 추정용. 화면 노출 금지 |
| `created_at` | DATETIME2 | 방문 시각 (KST) | 기간별 집계 |

인덱스: `(pv_path, created_at)`, `(created_at)`

## 3. 페이지 매핑 (pv_path → 화면 표시 이름)

**주요 페이지 (화면에서 항상 보여줄 것):**

| pv_path | 표시 이름 |
|---|---|
| `/` | 홈 (메인) |
| `/youth` | 청소년 코딩 |
| `/adult` | 성인 1:1 AI활용 |
| `/corporate` | 기업 AI교육 |
| `/autobiography` | AI자서전 출판 |
| `/campus` | 캠퍼스 |
| `/plaza` | 작품소개 |

※ `/plaza`는 특수: 작품소개는 다른 서버(game.kevinsaem.com)에 있어서, 랜딩 사이트 메뉴 클릭 시 `/plaza`를 거쳐 이동하며 그 시점에 1건 기록된다. 즉 **"작품소개 메뉴 클릭 수"**를 뜻한다.

**보조 페이지 (표에 있으면 함께 표시):** `/corporate-survey` 기업 설문, `/tuition/youth` 청소년 수강료, `/tuition/corporate` 기업 수강료, `/privacy` 개인정보처리방침, `/refund` 환불 규정

※ 매핑에 없는 새 경로는 경로 그대로 표시 (랜딩에 페이지가 추가될 수 있음).

## 4. 만들 화면 (요구사항)

관리자 사이드바에 **"접속 통계"** 메뉴 추가 (기존 관리자 메뉴와 동일한 관리자 인증 적용). 구성:

1. **요약 카드**: 오늘 / 최근 7일 / 최근 30일 / 전체 방문 수
2. **일별 방문 그래프** (최근 14~30일, 방문 없는 날은 0)
3. **페이지별 방문 표**: 위 주요 페이지 모두 — 오늘/7일/30일/전체 방문 수
4. **★ 페이지별 외부 유입경로** (이 화면의 핵심):
   - 페이지를 선택하면(탭/드롭다운) 그 페이지의 유입 출처별 방문 수를 보여준다
   - 유입 출처 = `pv_ref_host` 기준 (google.com, naver.com, instagram.com …), 빈 값은 **"직접 방문"**으로 표시
   - 자체 이동(`pv_ref_host`에 `kevinsaem` 포함)은 제외
5. **UTM 캠페인 성과 표**: `pv_utm_source`/`pv_utm_campaign` 별 방문 수 (기간 선택 가능하면 좋음) — 사장님이 광고 링크에 utm을 붙여 배포하면 여기서 캠페인별 반응이 보인다
6. **기기별 비율**: mobile / tablet / pc

데이터가 없을 때는 "아직 기록된 방문이 없습니다" 안내.

## 5. 참고 쿼리 (MS-SQL)

날짜 파라미터는 `'YYYY-MM-DD'` 문자열.

```sql
-- ① 페이지별 방문 수 (오늘/7일/30일/전체)
SELECT pv_path,
       SUM(CASE WHEN created_at >= @today THEN 1 ELSE 0 END) as cnt_today,
       SUM(CASE WHEN created_at >= @d7    THEN 1 ELSE 0 END) as cnt_7d,
       SUM(CASE WHEN created_at >= @d30   THEN 1 ELSE 0 END) as cnt_30d,
       COUNT(*) as cnt_total
FROM ek_PageView
GROUP BY pv_path
ORDER BY cnt_30d DESC;

-- ② 일별 방문 수
SELECT CONVERT(varchar(10), created_at, 23) as day, COUNT(*) as cnt
FROM ek_PageView
WHERE created_at >= @dfrom
GROUP BY CONVERT(varchar(10), created_at, 23)
ORDER BY day ASC;

-- ③ ★ 페이지별 외부 유입경로 (핵심 화면용)
SELECT pv_path, pv_ref_host, COUNT(*) as cnt
FROM ek_PageView
WHERE created_at >= @d30
  AND pv_ref_host NOT LIKE '%kevinsaem%'   -- 자체 이동 제외 ('' 직접 방문은 포함)
GROUP BY pv_path, pv_ref_host
ORDER BY pv_path, cnt DESC;
-- pv_ref_host = '' 은 화면에서 "직접 방문"으로 표시

-- ④ UTM 캠페인 성과
SELECT pv_utm_source, pv_utm_medium, pv_utm_campaign, COUNT(*) as cnt
FROM ek_PageView
WHERE created_at >= @d30 AND pv_utm_source <> ''
GROUP BY pv_utm_source, pv_utm_medium, pv_utm_campaign
ORDER BY cnt DESC;

-- ⑤ 기기별 비율
SELECT pv_device, COUNT(*) as cnt
FROM ek_PageView
WHERE created_at >= @d30
GROUP BY pv_device;

-- ⑥ (선택) 순방문자 추정 — 같은 IP+브라우저는 하루 1명으로 계산
SELECT COUNT(*) as unique_visitors FROM (
    SELECT DISTINCT pv_ip, pv_user_agent, CONVERT(varchar(10), created_at, 23) as day
    FROM ek_PageView
    WHERE created_at >= @dfrom
) t;
```

## 6. 주의사항

- `pv_ip`, `pv_user_agent`는 개인정보 성격 — **화면에 노출 금지** (집계 수치만).
- `ek_PageView`에는 **읽기만** 한다. 쓰기/스키마 변경은 랜딩 프로젝트 담당.
- 완성 검증 기준: 랜딩페이지를 브라우저로 몇 번 방문(일부는 `?utm_source=test&utm_campaign=check` 붙여서)한 뒤, 관리자 화면의 페이지별 수치·유입경로·UTM 표에 그대로 반영되는지 확인.

## 7. 사장님용 참고 — 마케팅 링크 만드는 법

광고/홍보물에 링크를 넣을 때 뒤에 utm 파라미터를 붙이면 캠페인별 반응이 구분되어 보인다:

```
https://booking.kevinsaem.com/youth?utm_source=instagram&utm_medium=ad&utm_campaign=2026_summer
https://booking.kevinsaem.com/adult?utm_source=blog&utm_medium=post&utm_campaign=review01
```

- `utm_source` = 어디에 (instagram, naver, blog, kakao …)
- `utm_medium` = 어떤 형태로 (ad 광고, post 게시글, message 메시지 …)
- `utm_campaign` = 무슨 캠페인인지 자유롭게 이름 붙이기
